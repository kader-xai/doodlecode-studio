# Writing a DoodleCode `.py` file by hand

You don't need the app to create a DoodleCode notebook. Every file is
plain Python with a handful of comment conventions — and a stable file
format you can `git diff` like any other source file.

This guide is the **shortest path from blank file to a polished
presentation deck**. The full spec lives in
[FILE_FORMAT.md](FILE_FORMAT.md); this is the friendlier walkthrough.

---

## 1. The minimal file

A file that opens in DoodleCode Studio at all just needs the cell
marker `# %%`:

```python
print("hello")

# %%
print("second cell")
```

One marker → two cells. The first cell is "everything before the first
marker". Each subsequent `# %%` starts a new cell.

---

## 2. Naming a cell (the **title bar**)

Add `title="..."` to the marker line:

```python
# %% title="Saying hello"
print("hello, world")

# %% title="Doing math"
print(2 + 2)
```

The title appears on the colored strip at the top of the cell. In the
app, double-click that strip to rename it inline.

---

## 3. Color and section kind

Two more knobs on the marker:

```python
# %% kind=function color=mint title="Compute area"
def area(r):
    return 3.14 * r * r
```

`color` accepts any of the named palette colors:

```
sky · mint · yellow · pink · peach · lavender · rose · cyan · violet
green · blue · orange · red · paper · white
```

`kind` is a free-text label (e.g. `function`, `loop`, `data`, `intro`).
If you omit `color`, one is picked from the kind. Both fields are
optional.

---

## 4. The callout — your explanation on the right

Add `# @explain:` lines right after the marker. Each line becomes one
sentence of the bubble body:

```python
# %% kind=function color=mint title="Compute area"
# @explain: Takes a radius and returns the area of a circle.
# @explain: math.pi is a built-in constant — about 3.14159.
import math

def area(r):
    return math.pi * r * r
```

What appears in the app:

- A **mint** strip on top of the cell saying "Compute area".
- A mint callout bubble on the right whose body is the two `@explain`
  lines joined with a single space.

Multiple `# @explain:` lines are just for line-wrapping in your editor —
the parser joins them. Internal multi-spaces ARE preserved (handy for
ASCII tables in callouts).

---

## 5. More callouts on the same cell

Open new bubbles with `# @callout`:

```python
# %% kind=loop color=peach title="Training loop"
# @explain: Repeat 100 times: predict → loss → backprop → step.
# @callout title="Aside · learning rate" color=lavender
# @explain: lr=0.01 is the step size. Smaller is more stable, slower.
# @callout color=rose
# @title: When to stop
# @explain: Watch validation loss, not training loss.
for epoch in range(100):
    ...
```

Inside a `# @callout` block, use either the inline form
(`# @callout title=...`) or follow-up `# @title:` / `# @color:` lines —
both work.

---

## 6. Embedding an image

`# @image:` adds a thumbnail to the callout (or, for text cells, inline
into the body). Use a data URL so the image travels with the file:

```python
# %% kind=function color=mint title="Architecture diagram"
# @explain: Two-layer NN: input → hidden(ReLU) → output(sigmoid).
# @image: data:image/png;base64,iVBORw0KGgoAAAA…
def model(x):
    ...
```

In the app, the easiest way is **✎ Callout** → 🖼 Choose… — it does the
base64 encoding for you and enforces a 1.5 MB ceiling.

---

## 7. Plain markdown cells (intros, headings, recaps)

```python
# %% [markdown]
# ## What you'll learn
# - foo
# - bar
# - baz
```

Each line is prefixed with `# ` (or just `#` for blank lines). The body
is rendered as markdown: `#` `##` `###` headings, `-`/`*` bullets,
`**bold**`, `` `inline code` ``.

---

## 8. Slide-style text boxes (v0.4+)

Mix `[markdown]` with the same `color` / `title` / `image` directives
and you get an editable presentation slide:

```python
# %% [markdown] color=sky title="Welcome"
# @image: data:image/png;base64,iVBORw0KGgoAAAA…
# # Workshop intro
# Today we'll explore Python in three slides.
```

In the app this renders as a text box with:
- A colored title bar on top ("Welcome").
- The image displayed inline above the text.
- The markdown body rendered as styled text.
- **📝 Edit** and **✎ Callout** buttons in the header for further edits.

---

## 9. Tags (cosmetic)

```python
# @tags: math, geometry
```

Comma-separated. They're stored in the cell meta but not rendered
anywhere visible yet — useful for downstream tooling.

---

## 10. Format-version banner

When the app saves a file it writes this at the top:

```text
# doodlecode format-version: 2
# Generated by DoodleCode Studio. Safe to edit; see docs/FILE_FORMAT.md.
```

You don't have to write it by hand — old files without it are
auto-detected as v1 and still parse. But if you want forward compat,
include it.

## A complete copy-paste template

```python
# doodlecode format-version: 2

# %% [markdown]
# # My lesson
# Short intro paragraph.

# %% [markdown] color=sky title="Goal"
# @image: data:image/png;base64,…           # optional
# # What you'll learn
# - one
# - two
# - three

# %% kind=intro color=sky title="Hello, world"
# @explain: print sends text to stdout.
# @explain: Try changing the message and re-running.
print("Hello, world!")

# %% kind=function color=mint title="Compute area"
# @explain: Wraps the formula so we don't repeat it.
# @callout title="Why a function?" color=lavender
# @explain: Functions make code reusable AND nameable.
import math

def area(r):
    return math.pi * r * r


print(area(3))

# %% [markdown]
# ## Recap
# - **print** for output
# - **def** for reusable logic
```

Save as `my_lesson.py`. Open it with **📂 Open**. Hit **🎬 Present**.
That's it.

---

## Tips

- **Edit the title** inside the app by **double-clicking** the colored
  strip on any cell.
- Use the **📐 Auto-Space [Presentation]** toolbar button to spread
  cells one-per-viewport for a clean presentation. The same button
  swaps to **🔗 Together** to bring them back for editing.
- Outside the app, treat the file like any other `.py` — black/ruff/
  isort all work, `python my_lesson.py` runs it top to bottom.
- Editors with the VS Code "Jupyter Interactive Window" extension
  recognise the same `# %%` markers, so DoodleCode files open as Jupyter
  cells there too (without the doodle visuals, of course).

For the full grammar, including escape rules and round-trip guarantees,
see [FILE_FORMAT.md](FILE_FORMAT.md).
